Add UploadThing integration with auto-WebP conversion for article images

- Install uploadthing & @uploadthing/react packages
- Add FileRouter (core.ts) with 2 endpoints:
  - featuredImage: 1 file, max 4MB (for blog article hero images)
  - galleryImage: up to 8 files, max 4MB each (for future gallery feature)
- Create route handler at /api/uploadthing (GET config, POST upload)
- Re-export UploadButton & UploadDropzone components with typed FileRouter
- Add ImageUploader wrapper component (CMS) with:
  - Drag & drop UI via @uploadthing/react
  - Preview after upload
  - Alt text input for SEO & accessibility
  - WebP badge indicator (confirms CDN auto-conversion)
  - Manual URL fallback (paste external image URL)
  - Remove/replace image button
- Update article new/edit forms to use ImageUploader instead of plain URL input
- Update next.config.mjs to allow utfs.sh & image.ufs.sh remote patterns
- Update .env.example with all 3 UploadThing env vars (TOKEN, SECRET, APP_ID)

Auto WebP conversion happens at UploadThing's CDN level (image.ufs.sh):
- All uploaded JPG/PNG automatically served as WebP to supported browsers
- Original kept for non-WebP clients
- Significantly reduces file size (~30-50%) for faster page load
- No additional server processing needed
